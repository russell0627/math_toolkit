# BUREAU OF MATHEMATICAL STANDARDS
## ALGEBRAIC ALIGNMENT PROTOCOL-9

### 1. OBJECTIVE
To isolate and identify the value of an unknown variable (typically represented by `x`) within a linear equation.

### 2. FUNDAMENTAL PRINCIPLE
**The Law of Reciprocity:** Any operation applied to one side of the equation MUST be applied to the other side to maintain "Bureaucratic Balance."

### 3. THE ISOLATION PROCEDURE
Follow these steps to attain variable transparency:

*   **Step A: Consolidation (Manual)**
    Combine numerical constants and like terms. The Algebra Assistant requires an explicit **SIMPLIFY EXPRESSION** command (Auto-Fix icon) to merge terms after an operation manifestation.
*   **Step B: Inverse Addition**
    If a number is added to `x`, subtract it from both sides. If it is subtracted, add it.
*   **Step C: Inverse Multiplication**
    If `x` is multiplied by a coefficient, divide both sides by that coefficient.
*   **Step D: Variable Centralization**
    If the variable term manifests on both sides (e.g., `3x = x + 10`), you must migrate all variable instances to a single side. Apply the inverse of the smaller variable term to both sides.
*   **Step E: Inequality Protocol**
    Support for `<`, `>`, `<=`, and `>=` is active.
    *   **Relation Flipping**: The system automatically flips the relation symbol if you multiply or divide by a negative value.
*   **Step F: Proportional Realignment (Cross-Multiplication)**
    If the equation resides in a fractional/proportional state (e.g., `a/b = c/d`), execute the **CROSS-MULTIPLICATION PROTOCOL** to linearize the system. Logic: `a * d = c * b`.
*   **Step G: System Recalibration**
    Use the **RECALIBRATE** button (Refresh icon) in the AppBar to reset the tool to its baseline state, clearing all history and current node data.
*   **Step H: Radical Realignment (Square Root)**
    To resolve powers (e.g., `x^2 = 25`), apply the **SQRT** operation from the transformation dropdown. This applies a square root to both sides.

### 4. COMMAND EXAMPLES
*   **Case 1: 2x + 10 = 20**
    1.  Apply `- 10` to both sides.
    2.  Result: `2x = 10`.
    3.  Apply `/ 2` to both sides.
    4.  Result: `x = 5`.
*   **Case 2: 3x = x + 10** (Variables on Both Sides)
    1.  Apply `- x` to both sides.
    2.  Result: `2x = 10`.
    3.  Apply `/ 2` to both sides.
    4.  Result: `x = 5`.
*   **Case 3: x/5 = 10/2** (Proportional Realignment)
    1.  Execute **CROSS-MULTIPLICATION PROTOCOL**.
    2.  Result: `2 * x = 50`.
    3.  Apply `/ 2` to both sides.
    4.  Result: `x = 25`.
*   **Case 4: x^2 = 25** (Radical Realignment)
    1.  Select **SQRT** from the operation dropdown.
    2.  Hit **Enter** or tap **APPLY**.
    3.  Result: `(x^2)^0.5 = (25)^0.5`.
    4.  Tap **SIMPLIFY** (Auto-Fix).
    5.  Result: `x = 5`.

### 5. SYSTEM COMPATIBILITY
The Bureau's Algebra Assistant automatically normalizes implicit notation.
*   `2x` is understood as `2 * x`.
*   `x(x+1)` is understood as `x * (x + 1)`.

> [!IMPORTANT]
> Failure to apply operations to both sides results in "Alignment Drift." Use the **Undo** button to revert any unsanctioned mathematical drift.
